<form></form>

Etiqueta de apertura y cierre de un formulario de una página web. El resto de etiquetas de formulario deben ir siempre recogidas entre estas etiquetas de apertura y cierre de formulario.

Uso:

<form></form>

Atributos:

accept-charset | action | autocomplete | enctype | method | name | novalidate | rel | target

Etiqueta de Cierre: Requerida.

Posicion: Dentro de la etiqueta <body>

Elementos que puede contener:

  • <input>
  • <textarea>
  • <button>
  • <select>
  • <option>
  • <optgroup>
  • <fieldset>
  • <label>

Codigo:

<form target="_self" action="datos.php" method="post">
nombre:<input type="text" name="nombre" id="nombre" required>
<button type="submit">Guardar</button>
</form>

action

action este atributo especifica dónde enviar los datos del formulario cuando se envía un formulario.

Variaciones de uso
  • Una URL absoluta: apunta a otro sitio web (action="http://www.pratapo.com/desdecasa.html")
  • Una URL relativa: apunta a un archivo dentro de un sitio web (action="guarda.php")

Nombre:<input type="text" name="nombre" required>
Apellido:<input type="text" name="apellido" required>
<input type="submit" value="Enviar">
</form>

autocomplete

autocomplete este atributo especifica si un campo de entrada debe tener habilitado el autocompletado.

Autocompletar permite que el navegador prediga el valor. Cuando un usuario comienza a escribir en un campo, el navegador debe mostrar opciones para completar el campo, en función de los valores escritos anteriormente.

Variaciones de uso
  • on
  • off

Codigo:

<form autocomplete="on">
telefono:<input type="text" name="telefono" required>
<input type="submit" value="Enviar">
</form>

enctype

enctype este atributo especifica cómo se deben codificar los datos del formulario al enviarlos al servidor.

enctype solo se puede usar si method="post".

Variaciones de uso
  • application/x-www-form-urlencoded Defecto. Todos los caracteres se codifican antes de enviarse (los espacios se convierten en símbolos "+" y los caracteres especiales se convierten en valores ASCII HEX).
  • multipart/form-data Este valor es necesario si el usuario cargará un archivo a través del formulario
  • text/plain Envía datos sin ningún tipo de codificación. No recomendado

Codigo:

<form action="datos.php" enctype="multipart/form-data" method="post">
correo:<input type="email" name="correo" required>
avatar:<input type="file" name="archivo">
<input type="submit" value="Enviar">
</form>

method

method este atributo especifica cómo enviar datos de formulario (los datos de formulario se envían a la página especificada en el atributo action).

Variaciones de uso

  • get recomendado solo para datos que no requieran seguridad.

    Consideraciones

    Agrega datos de formulario a la URL en pares de nombre/valor.

    La longitud de una URL es limitada (alrededor de 3000 caracteres).

    ¡Nunca use GET para enviar datos confidenciales! (será visible en la URL).

    Útil para envíos de formularios en los que un usuario desea marcar el resultado.

    GET es mejor para datos no seguros, como cadenas de consulta en Google.

  • post recomendado para datos que requieran seguridad y resguardo de la información.

    Consideraciones

    Agrega datos de formulario dentro del cuerpo de la solicitud HTTP (los datos no se muestran en la URL).

    No tiene limitaciones de tamaño.

    Los envíos de formularios con POST no se pueden marcar.

Codigo:

<form action="datos.php" method="get">
correo:<input type="email" name="correo" required>
clave:<input type="password" name="clave" required>
<input type="submit" value="Enviar">
</form>

rel

rel este atributo especifica la relación entre el documento actual y el documento vinculado.

Variaciones de uso
  • external especifica que el documento al que se hace referencia no forma parte del sitio actual.
  • help enlaces a un documento de ayuda.
  • license enlaces a la información de derechos de autor del documento.
  • next el siguiente documento en una selección.
  • nofollow es aquel que impide que fluya la popularidad (trafico) de la página origen del enlace hacia la página destino del enlace. Esto quiere decir que la página que recibe el enlace no se beneficia en absoluto de la popularidad ni credibilidad de la página origen del enlace.
  • noopener es un atributo HTML que impide que la página de apertura obtenga algún tipo de acceso a la página original.
  • noreferrer este atributo evita pasar la información de referencia al sitio web de destino al eliminar la información de referencia del encabezado HTTP.

Codigo:

<form action="datos.php" method="post" rel="noopener noreferrer">
correo:<input type="email" name="correo" required>
clave:<input type="password" name="clave" required>
<input type="submit" value="Enviar">
</form>
